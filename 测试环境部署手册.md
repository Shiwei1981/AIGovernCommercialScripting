# 测试环境部署手册（开发机 Linux，人工 UI 自测）

> 适用场景：在开发服务器（Linux）上，直接启动当前代码进行**人工界面自测**。  
> 本手册不创建任何 Azure 云资源，仅做存在性与连通性校验。

## 1. 前置条件

1. 已在 Linux 开发机获取本仓库代码目录。
2. `.env.local` 已存在且包含真实测试环境变量（非占位值）。
3. 机器可访问：
   - Azure SQL Database
   - Azure OpenAI Endpoint
   - Google News RSS 与新闻网页

## 2. 本手册使用的脚本

本手册仅使用以下脚本：

1. `测试环境部署一键脚本.sh`（一键部署并启动测试环境）
2. `scripts/sql/verify_ai_generation_log.sql`（验证 AI 日志入库）

其中 `测试环境部署一键脚本.sh` 会**自动加载 `.env.local`**，并在运行前检查：
- 必填变量是否缺失
- 是否存在占位值（如 `your-*`、`<...>`）

## 3. Step-by-step 部署与启动

### Step 1：进入项目目录

```bash
cd /home/shiwei/AIGovernDemoCode/AIGovernCommercialScripting
```

### Step 2：执行一键部署脚本

```bash
bash ./测试环境部署一键脚本.sh
```

脚本会自动完成：
1. 加载 `.env.local`
2. 校验必填环境变量与占位值
3. 创建/激活 `.venv` 并安装依赖
4. 校验 Azure SQL / Azure OpenAI / Google RSS 连通性
5. 校验业务表结构是否符合代码使用要求
6. 执行 `scripts/sql/001_ai_generation_log.sql`，对 `AIGenerationLog` 做结构对齐
7. 启动应用服务并附着到当前终端（实时输出），同时写入日志并记录 PID

脚本完成启动检查后不会自动退出，会持续显示应用输出；在该终端按 `Ctrl+C` 可停止服务。

### Step 3：确认服务已启动

脚本成功后会输出：
- `URL: http://127.0.0.1:${WEB_APP_PORT}`
- `Log: logs/test-env-app.log`
- `PID: <pid>`

也可手工检查：

```bash
curl -sS http://127.0.0.1:${WEB_APP_PORT}/api/session
```

## 4. 人工 UI 自测步骤（按业务流程）

在浏览器打开：

```text
http://127.0.0.1:${WEB_APP_PORT}
```

按以下顺序执行：

1. 未登录进入页面，确认业务操作不可用，仅可登录。
2. 点击 **Sign in with Entra ID** 完成同 tenant 用户登录。
3. 登录后确认页面自动显示产品列表（无需手动点 Load）。
4. 输入客户描述，点击 **Query Target Customers**，确认返回客户列表与生成 SQL。
5. 选择一个客户，点击 **Customer Analysis**，确认订单历史与分析结果展示。
6. 选择一个产品，点击 **Product Trend Search**，确认展示新闻列表、趋势总结、来源信息。
7. 点击 **Generate Ad Copy**，确认广告文案展示。
8. 点击 **Reset**，确认页面回到登录后默认状态（产品列表已显示，流程中间状态已清空）。

## 5. 错误处理验证（人工）

1. 不登录直接调用业务动作：应在 UI 显示错误且接口返回 401。
2. 趋势检索场景下若有网页抓取失败：应在 UI 看到失败提示，同时服务日志中有错误记录。
3. 若有效正文率低于阈值：应看到“趋势证据不足”和 `fetch_errors`。

查看日志：

```bash
tail -f logs/test-env-app.log
```

> 若脚本正在运行，当前终端已经实时显示服务输出；`tail -f` 主要用于另开终端查看。

## 7. 停止服务

在运行脚本的终端按 `Ctrl+C` 即可停止服务。

如需在其他终端停止，可使用：

```bash
kill "$(cat .test_env_app.pid)"
rm -f .test_env_app.pid
```

## 8. 常见失败与处理

1. 脚本报 `Missing required environment variables`  
   - 说明 `.env.local` 缺字段或值为空，补齐后重跑。

2. 脚本报 `Placeholder-like values detected`  
   - 说明 `.env.local` 里还有占位值，替换成真实值后重跑。

3. 脚本报 Azure SQL / OpenAI / RSS 校验失败  
   - 说明测试环境依赖不可达或凭据错误，修正 `.env.local` 后重跑。
