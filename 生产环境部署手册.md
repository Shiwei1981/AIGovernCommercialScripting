# 生产环境部署手册

> 适用场景：把当前仓库代码发布到**已存在**的 Azure Linux Web App 生产环境。  
> 本手册和配套脚本**不会创建任何 Azure 云资源**；如果资源不存在，脚本会直接报错并停止。  
> 所有 shell 脚本都会**自动加载仓库根目录的 `.env.local`**，并在执行前检查缺失值与占位值。  
> 脚本内部会自动切换到仓库根目录，因此可以在仓库根目录执行，也可以通过绝对路径直接调用脚本。

## 1. 本手册使用的脚本

| 脚本 | 作用 |
| --- | --- |
| `生产环境部署一键脚本.sh` | 按顺序串行执行完整生产部署 |
| `scripts/deploy/azure/validate_resources.sh` | 校验 Azure 资源组、App Service Plan、Linux Web App、自定义容器配置、ACR、域名绑定，以及 Azure SQL 私网 DNS 前置条件 |
| `scripts/deploy/azure/verify_db_schema.sh` | 校验生产数据库结构，自动对齐 `dbo.AIGenerationLog`，并验证 SQL / OpenAI / RSS |
| `scripts/deploy/azure/configure_app_settings.sh` | 把生产环境运行参数写入 Azure Web App App Settings |
| `scripts/deploy/azure/deploy_webapp_package.sh` | 基于 `Dockerfile` 构建容器镜像、推送到现有 ACR，并把 Azure Web App 切换到新镜像 |
| `scripts/deploy/azure/smoke_test_webapp.sh` | 对已部署站点做 HTTP 探活与鉴权边界检查 |
| `scripts/sql/001_ai_generation_log.sql` | `AIGenerationLog` 的幂等结构对齐 SQL |

脚本会要求以下变量非空：

### 2.3 生产覆盖项（可选）

```text

PROD_MOCK_AUTH_ENABLED      # 默认 false

PROD_CONTAINER_IMAGE_TAG    # 不填时自动使用 VERSION-UTC时间戳
```

## 3. 生产部署前说明

1. `SalesLT.*` 是项目既有业务基表，脚本会按代码和设计做**严格一致性校验**。
2. `dbo.AIGenerationLog` 是项目治理表，允许脚本自动结构对齐；脚本内部会执行 `scripts/sql/001_ai_generation_log.sql`。
3. 如果 `SalesLT.*` 与设计/代码不一致，脚本会停止部署并输出差异；**不会**在生产库里擅自重塑业务基表，因为当前项目设计明确把可自动变更的结构限定在 `AIGenerationLog`。
4. 当前生产环境是 **Azure Linux Custom Container Web App**，部署方式是“构建镜像 -> 推送到现有 ACR -> 切换 Web App 镜像”，而不是 zip 包源码发布。
5. 镜像构建上下文由仓库根目录的 `Dockerfile` 与 `.dockerignore` 控制；`.env.local` 不会进入镜像上下文。

## 4. Step-by-step 手工部署

### Step 1：进入仓库目录

```bash
cd /home/shiwei/AIGovernDemoCode/AIGovernCommercialScripting
```

### Step 2：赋予脚本执行权限

```bash
chmod +x \
  ./生产环境部署一键脚本.sh \
  ./scripts/deploy/azure/common.sh \
  ./scripts/deploy/azure/validate_resources.sh \
  ./scripts/deploy/azure/verify_db_schema.sh \
  ./scripts/deploy/azure/configure_app_settings.sh \
  ./scripts/deploy/azure/deploy_webapp_package.sh \
  ./scripts/deploy/azure/smoke_test_webapp.sh
```

### Step 3：单步执行资源校验

```bash
bash ./scripts/deploy/azure/validate_resources.sh
```

脚本会完成：

1. 加载 `.env.local`
2. 校验生产部署必填变量和占位值
3. 使用 `.env.local` 中的部署服务主体登录 Azure CLI
4. 校验资源组、App Service Plan、Linux Web App 是否存在
5. 校验 Web App 当前已配置为自定义容器，且使用托管身份从指定 ACR 拉取镜像
6. 校验现有 ACR 是否存在，且 login server 与 `.env.local` 一致
7. 如果 Azure SQL `Public Network Access=Disabled`，额外校验 Web App 集成 VNet 已链接到 `privatelink.database.windows.net`
8. 校验 `APP_BASE_URL` 域名是否已绑定到目标 Web App

### Step 4：单步执行数据库与外部依赖校验

```bash
bash ./scripts/deploy/azure/verify_db_schema.sh
```

脚本会完成：

1. 加载 `.env.local`
2. 准备本地 `.venv` 并安装 `requirements.txt`
3. 连通 Azure SQL
4. 校验 `SalesLT.*` 必需表/列/关键数据类型
5. 执行 `scripts/sql/001_ai_generation_log.sql`，自动对齐 `dbo.AIGenerationLog`
6. 校验 Azure OpenAI 调用可用
7. 校验 Google News RSS 与至少一个新闻正文抓取可用

### Step 5：写入生产环境 App Settings

```bash
bash ./scripts/deploy/azure/configure_app_settings.sh
```

脚本会把 `.env.local` 中的真实值写入 Azure Web App，包括：

1. `APP_ENV=production`（或 `PROD_APP_ENV`）
2. Web 端口、OIDC、Azure SQL、Azure OpenAI、Google RSS 等运行参数
3. `WEBSITE_DNS_SERVER=168.63.129.16`
4. `SCM_DO_BUILD_DURING_DEPLOYMENT=false`
5. `ENABLE_ORYX_BUILD=false`
6. 启动命令（默认 `python -m app.main`）

### Step 6：发布当前代码

```bash
bash ./scripts/deploy/azure/deploy_webapp_package.sh
```

脚本会完成：

1. 读取 `VERSION`，自动生成镜像 tag（若 `PROD_CONTAINER_IMAGE_TAG` 为空，则使用 `VERSION-UTC时间戳`）
2. 基于仓库根目录 `Dockerfile` 调用 `az acr build` 在现有 ACR 中构建并推送镜像
3. 通过 `az webapp config container set` 把现有生产 Web App 切换到新镜像
4. 重启 Web App

### Step 7：执行生产探活

```bash
bash ./scripts/deploy/azure/smoke_test_webapp.sh
```

探活通过条件：

1. `GET /` 返回 `200`
2. `GET /api/session` 返回 `200` 且 `authenticated=false`
3. `GET /api/products/bootstrap` 返回 `401`

## 5. 一键部署

如无需逐步排障，直接执行：

```bash
bash ./生产环境部署一键脚本.sh
```

该脚本会按以下顺序串行执行：

1. 校验 `.env.local`
2. 准备本地 Python 运行时
3. 校验 Azure 资源
4. 校验数据库并对齐 `AIGenerationLog`
5. 写入 App Settings
6. 构建并发布容器镜像，启动生产站点
7. 执行生产探活

浏览器打开：

```text

```

按以下业务流程验证：

1. 未登录访问首页，确认只能登录，业务动作不可用。
2. 点击 **Sign in with Entra ID**，确认可跳转并完成同 tenant 登录。
3. 登录后确认产品列表自动加载。
4. 输入客户描述并执行 **Query Target Customers**，确认返回客户结果与生成 SQL。
5. 选择客户执行 **Customer Analysis**，确认订单历史与 AI 分析返回。
6. 选择产品执行 **Product Trend Search**，确认新闻列表、趋势总结、来源链接可用。
7. 点击 **Generate Ad Copy**，确认广告文案生成成功。
8. 点击 **Reset**，确认页面恢复登录后默认状态。
9. 在数据库中检查 `dbo.AIGenerationLog` 有新增记录。

## 7. 常见失败与处理

1. 报 `Missing required environment variable(s)`  
   - 说明 `.env.local` 仍缺真实值；补齐后重跑。

2. 报 `Placeholder-like values detected`  
   - 说明 `.env.local` 仍有占位值；替换成真实生产值后重跑。

3. 报 Azure 资源不存在  
   - 说明生产资源还没准备好，或 `.env.local` 指向了错误资源；本脚本不会创建资源。

4. 报 `SalesLT` 结构不一致  
   - 说明生产库不是项目预期的既有业务库；先修正数据库，再重跑部署。

5. 报 OpenAI / RSS 校验失败  
   - 说明生产依赖不可达或凭据有误；修正 `.env.local` 后重跑。

6. 报 ACR build 失败  
   - 说明 `Dockerfile`、基础镜像拉取、Python 依赖安装或 ACR 构建权限存在问题；先看 `az acr build` 输出，再修正后重跑。

7. 报容器启动失败或探活失败  
   - 说明镜像已推送但 Web App 容器未正常启动；优先查看 Web App 容器日志，再核对启动命令、应用监听端口、运行时依赖和 App Settings。
